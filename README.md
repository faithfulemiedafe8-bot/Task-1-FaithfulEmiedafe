# Project 1 - Empathy Map

## Overview

This project was completed as part of the DecodeLabs UI/UX Design Internship (Batch 2026).

The objective of this exercise was to understand the target users before moving into interface design. The scenario focused on a fitness startup that wants to create a fitness application for busy office workers who frequently claim they do not have enough time to exercise.

Rather than jumping directly into wireframes or screens, the goal was to explore the user's motivations, behaviors, frustrations, and underlying challenges.

## Project Goal

The purpose of this project was to determine whether "lack of time" is the real problem preventing users from exercising, or if deeper factors such as motivation, habits, decision fatigue, and accountability play a larger role.

## What I Did

- Created a user persona (Chidi, 29, Software Engineer)
- Identified user goals, motivations, and pain points
- Developed a complete empathy map
- Analyzed what the user says, thinks, does, and feels
- Explored underlying reasons for workout avoidance, including decision fatigue and guilt from missed routines

## Key Insight

One of the main findings from this exercise was that the user's challenge is not simply a lack of time. Factors such as mental exhaustion after work, inconsistent routines, and feeling overwhelmed by fitness commitments often have a greater impact on behavior than schedule constraints alone.

## Why This Matters

Understanding users before designing solutions helps ensure that design decisions are based on real user needs rather than assumptions.

Without this research phase, it would be easy to create another generic fitness application that fails to address the actual barriers preventing users from maintaining healthy habits.
